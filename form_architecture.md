# Формируем архитектуру
1. Схемы создания объектов - объекты будут создаваться в начале игры без дополнительной инфраструктуры.  
2. Схемы обарботки событий - код будет крутиться в цикле. Необходимые методы будут вызываться из State. Интерфейс будет передаваться по DI в состояние отрисовки. Остальные игровые состояния будут напрямую взаимодейсвтовать с игровым полем игроком и статистикой.
3. Использование технологий - для отображения консоли будет использовать библиотека Textual.
4. Тесты https://github.com/mileoa/third_oop/tree/main/src/tests  

Также удалил некотрые команды и запросы из АТД во время написания тестов, так как понял что они особо не нужны.
