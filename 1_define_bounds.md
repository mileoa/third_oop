# Определение границ системы
## Что будет включено в систему и что нет
В сисетему необходимо включить отображение игроку различной информации о текущем состоянии игры через интерфейс, возможность менять состояние игрового поля как по инициативе игрока, так и по инициативе системы, игровая механика 3 в ряд.  
В систему нет необходимости включать хранение информации о текущем состоянии игры для запуска с того же места при перезапуске программы.  
## Главные подсистемы
Интерфейс, игровое поле, игрок, игровая логика.
## Пользовательские метафоры
Игровое поле - матрица 8x8 или NxM, которая хранит в себе элементы игрового поля.  
Специальный бонус - специальный эффект, который изменят поведение игры. Специальным бонусом можем быть удаление всех элементов определенного типа, удаление всего ряда и/или всей строки.  
Комбинация - расположение 3 игровых элементов одинакового типа по вертикали или горизонтали на игровом поле.  
Бонусы - сложные комбинации, например, расположение элементов в виде плюса.  
Интерфейс - то что видит пользователь, на нем отображаются ошибки и игровое поле.  
Ввод - возмонжость игрока взаимодействовать с ситемой через ввод специальных команд или координат элементов для смены мест.  
Очки - количество очков, которые пользователь набрал за удаление комбинаций.  
История - история ходов игрока.  
Статистика игры - общее количество очков, ходов игрока по завершении игры.  
## Функциональность
- Генерация случайного поля без начальных комбинаций  
- Перестановка элементов по координатам  
- Обнаружение и удаление комбинаций  
- Падение элементов  
- Генерация новых элементов сверху   
- Подсчет очков  
- Проверка наличия возможных ходов  
- Обработка ошибок ввода  
- Вывод статистики  
- Перезапуск/завершение игры  
- Применение специальных бонусов  
- Ведение истории  
## Библиотеки повторного использования
- Парсер консольных команд  
- Генератор случайных элементов  
- Поисковик шаблонов в матрице  
- Матрица  
- Консольный интерфейс отображения
